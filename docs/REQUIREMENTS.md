# AI秘書アプリ 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
AI秘書（AI Secretary）

### 1.2 目的
様々なAIツールを使用しなくても、一つのアプリケーションで日常業務の効率化を実現する統合型AI秘書アプリの開発

### 1.3 ターゲットユーザー
- 個人のビジネスパーソン
- ルーティン業務の効率化を求める方

## 2. 機能要件

### 2.1 タスク管理機能

#### 機能詳細
- Todoリスト形式でのタスク管理
- タスクのCRUD操作（作成・読取・更新・削除）
- 優先度設定（高・中・低）
- カレンダー連携による期限管理
- 期限2日前のアラート通知

#### データ構造
- タスク名
- 説明文
- 優先度
- 期限日時
- ステータス（未完了/完了）
- 作成日時
- 更新日時

### 2.2 文章校正機能

#### 機能詳細
- メール文章や報告書の校正
- 誤字脱字チェック
- 文法チェック
- 文体改善提案
- 構成提案
- ユーザーの文体学習・分析機能
- 学習した文体での校正

#### 処理フロー
1. ユーザーが文章を入力
2. OpenAI APIで文章解析
3. ユーザーの過去の文体パターンと照合
4. 校正案を提示
5. 校正履歴を保存（文体学習用）

### 2.3 議事録作成機能

#### 機能詳細
- 音声ファイルのアップロード
- 自動文字起こし
- 議事録フォーマットでの自動生成
- 議事録の保存・管理

#### 議事録フォーマット
- 議論された内容
- 決定事項
- ネクストアクション

#### 対応音声形式
- mp3
- wav
- m4a
- 最大ファイルサイズ: 5GB

### 2.4 リサーチ機能

#### 機能詳細
- テーマ入力による自動リサーチ
- ChatGPTによるプロンプト最適化
- Perplexity APIでのWeb検索実行
- 検索結果の要約
- 出典情報の一覧表示

#### 処理フロー
1. ユーザーがテーマを入力
2. OpenAI APIでリサーチ用プロンプトを生成
3. Perplexity APIで検索実行
4. 結果を要約して表示
5. 出典リンクを一覧表示

### 2.5 Googleカレンダー連携

#### 機能詳細
- カレンダーへの予定追加
- 予定の編集
- 予定の削除
- タスク管理との連携（期限設定）

## 3. 非機能要件

### 3.1 認証・セキュリティ
- Google OAuth 2.0による認証
- セッション管理
- APIキーの安全な管理
- HTTPSによる通信暗号化

### 3.2 パフォーマンス
- ページ読み込み: 3秒以内
- API応答時間: 5秒以内
- 音声ファイル処理: ファイルサイズに応じて非同期処理

### 3.3 可用性
- Vercelでのホスティング
- 99.9%のアップタイム目標

### 3.4 データ管理
- Supabaseでのデータ永続化
- 自動バックアップ
- ユーザーごとのデータ分離
- データ保存期間: **永久**（自動削除なし）
- データエクスポート: **なし**

### 3.5 UI/UX要件
- Apple風のモダンでクリーンなデザイン
- レスポンシブデザイン（デスクトップ優先）
- 直感的な操作性
- ライトモードのみ（ダークモード非対応）

## 4. 技術スタック

### 4.1 フロントエンド
- Next.js（App Router）
- React 19
- TypeScript
- Tailwind CSS v4

### 4.2 バックエンド
- Next.js API Routes
- Supabase（PostgreSQL）

### 4.3 外部API
- OpenAI API（文章校正、議事録作成、プロンプト生成）
- Perplexity API（Web検索）
- Google Calendar API（カレンダー連携）
- AssemblyAI API（音声文字起こし）

### 4.4 インフラ
- Vercel（ホスティング）
- Supabase（データベース、認証）

> **実装リポジトリのバージョン（2026-05）:** `secretary-platform` は Next.js **16.2.6** / React **19.2.4** でスキャフォールド済み。要件上の App Router・TypeScript・Tailwind v4 は満たす。

## 5. 制約事項

- 無料プランのみ（課金機能なし）
- シングルユーザー利用（マルチテナント非対応）
- データエクスポート機能なし
- ダークモード非対応
- モバイルアプリ版なし（Webアプリのみ）
- 認証: Googleアカウントのみ（メール/パスワード不要）

## 6. 成功基準

- 全5機能が統合された状態で動作
- Google認証によるログインが可能
- 各APIとの連携が正常に動作
- ユーザーデータが永続的に保存される
- Apple風のUIデザインが実装されている

## 7. リスクと対策

### リスク
- API利用料金の増大
- 音声ファイル処理の負荷
- 外部APIの仕様変更

### 対策
- APIコール数の最適化
- キャッシュの活用
- 非同期処理の実装
- エラーハンドリングの徹底
